# ИС ИКД — веб-интерфейс и API

Информационная система инженерно-конструкторской документации с назначением ответственных лиц и Node.js + SQLite бэкендом.

## Запуск

```bash
cd server
npm install
npm run seed
npm start
```

Откройте в браузере: **http://localhost:3000/index.html**

## Демо-учётные записи (пароль `123`)

| Логин | Роль |
|-------|------|
| `chief_engineer` | Главный инженер |
| `tech_dept_head` | Начальник технического отдела |
| `project_chief1` | ГИП |
| `designer_ivanov` | Инженер-проектировщик |

## Логика назначений

1. **Главный инженер** / **Начальник технического отдела** — создают проекты и **обязательно назначают ГИП** (главного инженера проекта).
2. **ГИП** — ответственное лицо за проект; на своих проектах выдаёт задания, выбирает **производственный отдел** и **инженера-проектировщика**.
3. **Главный инженер** — утверждает комплекты документации.
4. Регистрация сотрудников — раздел **«Сотрудники»** (для руководящих ролей).

## Структура

```
project/          — фронтенд (HTML/CSS/JS)
server/           — Express API + SQLite
server/data/      — файл базы ikd.db (создаётся автоматически)
```

## API (кратко)

- `POST /api/auth/login`
- `GET/POST /api/employees`
- `GET/POST /api/projects` — при создании поле `gipId` обязательно
- `GET/POST /api/tasks` — назначение отдела и исполнителя
- `PATCH /api/tasks/:id/assign` — переназначение исполнителя
- `GET /api/departments`
